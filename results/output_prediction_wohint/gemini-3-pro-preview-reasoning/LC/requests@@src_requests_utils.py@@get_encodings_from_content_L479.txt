The function `get_encodings_from_content` extracts encoding names from an HTML/XML string using three regular expressions. It returns a concatenation of the lists of matches found by each regex.

**1. Execution Analysis:**

*   **Input Content:**
    `content = "<meta http-equiv=\"Content-type\" content=\"text/html;charset=UTF-8\">"`

*   **Regex 1: `charset_re`**
    Pattern: `<meta.*?charset=["\\\']*(.+?)["\\\'>]` (Case Insensitive)
    *   This regex looks for `<meta`, followed by any characters, then `charset=`.
    *   In the input string: `<meta http-equiv="Content-type" content="text/html;charset=UTF-8">`
    *   It matches `<meta`.
    *   `.*?` consumes ` http-equiv="Content-type" content="text/html;`.
    *   `charset=` matches the literal `charset=` inside the `content` attribute.
    *   `["\\\']*` matches zero quotes (the next character is `'U'`).
    *   `(.+?)` captures characters non-greedily until a delimiter. It captures `UTF-8`.
    *   `["\\\'>]` matches the closing double quote `"` of the `content` attribute.
    *   **Match:** `'UTF-8'`.
    *   **Result:** `['UTF-8']`.

*   **Regex 2: `pragma_re`**
    Pattern: `<meta.*?content=["\\\']*;?charset=(.+?)["\\\'>]` (Case Insensitive)
    *   This regex looks for `content=`, optionally a quote, optionally a semicolon, and then *immediately* `charset=`.
    *   In the input string, after `content="`, we have `text/html;charset=`.
    *   The literal `text/html;` is present between the quote and `charset=`.
    *   The regex part `["\\\']*;?` does not allow for `text/html;`.
    *   **Result:** `[]` (No match).

*   **Regex 3: `xml_re`**
    Pattern: `^<\\?xml.*?encoding=["\\\']*(.+?)["\\\'>]`
    *   This regex searches for an XML declaration at the start of the string.
    *   The string starts with `<meta`.
    *   **Result:** `[]` (No match).

*   **Return Value:**
    The function returns `charset_re.findall(content) + pragma_re.findall(content) + xml_re.findall(content)`.
    Calculation: `['UTF-8'] + [] + []` = `['UTF-8']`.
