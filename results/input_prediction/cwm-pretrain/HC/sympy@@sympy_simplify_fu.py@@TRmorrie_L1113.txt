# Action Readme

Welcome to the Action project! This guide will help you set up and run the project seamlessly using the provided scripts.

## Prerequisites

Before you begin, ensure you have the following installed on your system:
- Git
- Bash shell
- Python 3.9 or later

## Setup and Execution Steps

### 1. Clone the Repository
First, clone the repository to your local machine:
```bash
git clone https://github.com/amaranja/action.git
cd action
```

### 2. Navigate to the scripts directory
Move into the scripts directory:
```bash
cd scripts
```

### 3. Install Requirements
Run the setup script to install the required Python packages:
```bash
bash setup.sh
```
This script will install necessary dependencies listed in requirements.txt.

### 4. Run the Main Script
To execute the main script, run:
```bash
bash main.sh
```
This will start the main application or process defined in main.sh.

### 5. Debugging
If you encounter any issues, you can check the logs generated by the setup and main scripts:
- Check setup.log for issues related to dependency installation.
- Check main.log for any errors during script execution.

### 6. Configuration
Before running the script, you might need to edit the config.yaml file to set up your project configurations.
The configuration file includes parameters for GPU utilization, CUDA settings, and other environment-specific settings.

### 7. Performance Tuning
To optimize GPU usage, ensure your system has the correct CUDA drivers installed and matches the required CUDA version specified in the setup script. You can modify the main script to adjust GPU resources as needed.

### 8. Clean Up
After running the scripts, you may clean up temporary files and log files if necessary by running:
```bash
rm -f setup.log main.log
```

## Common Issues and Solutions

- Issue: Missing CUDA or Python 3.9.
  - Solution: Install CUDA 11.6.0 or Python 3.9.12 as per the setup script requirements.

- Issue: Script fails due to missing packages.
  - Solution: Ensure all dependencies are installed by rerunning setup.sh.
