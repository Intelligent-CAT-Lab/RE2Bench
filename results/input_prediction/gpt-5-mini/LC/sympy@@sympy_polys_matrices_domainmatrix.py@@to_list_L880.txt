[THOUGHT]
DomainMatrix.to_list simply returns self.rep.to_list(). The SDM.to_list implementation creates a list-of-lists with dimensions given by M.shape and then fills entries from M.items() (row -> {col: element}). To produce the provided 2x2 output of Matrix objects, rep must represent a 2x2 sparse mapping where each (i,j) entry is the corresponding Matrix.

- shape must be (2, 2).
- rep.items() must provide all four entries so the zero from domain is never relied upon.
- Each entry should be the Matrix shown in the output.

Thus set rep as a mapping:
- row "0": col "0" -> Matrix([[1, 0],[0, 1]]), col "1" -> Matrix([[1, 2, 3],[3, 5, 4]])
- row "1": col "0" -> Matrix([[4, 2],[2, 3],[7, 5]]), col "1" -> Matrix([[1, 1, 1],[1, 1, 1],[1, 1, 1]])

domain can provide any zero (not used because all entries are present); use 0. args and kwargs are empty.
